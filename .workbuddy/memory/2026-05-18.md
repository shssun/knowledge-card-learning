## Bug 修复：卡片不保存 + 用户等级未接入（2026-05-18 晚）

### Bug1：生成卡片后重新进入需要重新生成
**根因**：`StudyPage.tsx` useEffect 依赖数组为 `[materialId, entryMode, term]`，但 session 复用逻辑调用的是 `setCurrentSession`（只改 `currentSessionId`，sessions 数组不变），导致 effect 不重跑，刷新后 `currentSessionId` 指向旧 session，但 effect 找不到该 session。

**修复**：把 `currentSessionId` 加入 useEffect 依赖数组 `sessions`。

### Bug2：设置里选了小白，但卡片仍有5个标题
**根因**：`userLevel` 存在 localStorage（`zhika-settings`），但 `cardService.generateCard` 和 `CARD_GENERATION_PROMPT` 完全没有读取等级参数，AI 永远生成全部字段。

**修复**：
- `ai-prompts.ts` 新增 `LEVEL_FIELD_CONFIG`，按等级控制生成字段：
  - 小白：`coreDefinition`（1个）
  - 初级：`coreDefinition` + `applicableScope`（2个）
  - 中级：+ `basicCase`（3个）
  - 高级/大师：全部（4个）
- `CARD_GENERATION_PROMPT` 加 `userLevel` 参数，prompt 中明确告知 AI 目标读者等级
- `Step2_GenerateCard` 从 localStorage 读取 `userLevel`，传给 `generateCard`
- 卡片详情弹窗按等级过滤展示标题
- 页面标题区显示当前等级说明

## Step4 融合卡片等级过滤 + 按钮修复（2026-05-18 深夜）

### 问题1：小白等级融合卡片仍展示全部4个维度
**根因**：`LEVEL_FIELD_CONFIG` 只控制了 Step2 标准卡片的生成和展示，Step4 融合卡片完全没接入等级过滤。

**修复**：
- `ai-prompts.ts` 新增 `FUSION_LEVEL_FIELD_CONFIG` 和 `FUSION_FIELD_LABELS`：
  - 小白：`personalizedDefinition`（1个）
  - 初级：`personalizedDefinition` + `refinedBoundary`（2个）
  - 中级：+ `practicalCases`（3个）
  - 高级/大师：全部4个（含 `deepLogic`）
- `Step4_FusionCard` 从 localStorage 读取 `userLevel`，按等级动态渲染融合卡片字段

### 问题2："下一步：输出评分"按钮灰色不可点
**根因**：按钮禁用条件为 `fusionCards.length !== generatedCards.length`，理论上单卡模式两边都是1应该可点，但实际可能是 session 数据不一致导致。

**修复**：改为 `!hasFusionCard`，只要有当前卡片的融合卡片就解锁按钮，更直接可靠。
