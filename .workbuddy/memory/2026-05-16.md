## Bug修复：资料中心"开始学习"按钮无响应

**问题**：`MaterialLibrary.tsx` 中的"开始学习"按钮没有绑定 `onClick` 事件，点击无任何反应。

**修复方案**：
1. `MaterialLibrary.tsx`：引入 `useNavigate` + `ROUTES`，给按钮添加跳转逻辑 `navigate(\`${ROUTES.STUDY}?materialId=${material.id}\`)`
2. `StudyPage.tsx`：引入 `useSearchParams`，从 URL 参数读取 `materialId`，传给 `createSession(materialId, [])`；添加 `materialId` 到 `useEffect` 依赖数组
3. `Step1_SelectContent.tsx`：从 session 读取 `materialId`，用 `useMemo` 计算预填充关键词；`handleNext` 复用已有 session 而非重复创建

**涉及文件**：
- `src/features/materials/MaterialLibrary.tsx`
- `src/pages/StudyPage.tsx`
- `src/features/study/Step1_SelectContent.tsx`

## Bug修复：学习中心"开始生成卡片"按钮无响应

**问题**：`StudyWizard.tsx` 中存在多处问题导致流程无法推进：
1. `currentStep` 从 store 解构但该字段不存在于 store 顶层，永远是 `undefined`
2. `setCurrentStep(step)` 只传了 1 个参数，store 签名是 `(sessionId, step)`
3. `completeSession()` 缺少 `sessionId` 参数
4. 硬编码路由字符串

**修复方案**：
1. `StudyWizard.tsx`：从 `getCurrentSession()` 获取 `session`，使用 `session?.currentStep`；所有 `setCurrentStep` 改为传两个参数 `(session.id, step)`
2. `Step5_OutputScore.tsx`：接口添加 `onBack: () => void`，`completeSession` 传入 `session.id`
3. 所有硬编码路由替换为 `ROUTES` 常量

**涉及文件**：
- `src/features/study/StudyWizard.tsx`
- `src/features/study/Step5_OutputScore.tsx`

## 全局路由规范修复

**问题**：多个文件使用 `window.location.href` 或硬编码 `navigate('/xxx')` 导致非 SPA 导航或路由不统一。

**修复文件**（共 5 个）：
- `src/features/review/FinalAssessment.tsx`：2处 `window.location.href` → `navigate(ROUTES.STUDY/ARCHIVE)`
- `src/features/review/ReciteMode.tsx`：`window.location.href = '/review'` → `navigate(ROUTES.REVIEW)`
- `src/features/review/ReviewQueue.tsx`：`window.location.href = '/study'` → `navigate(ROUTES.STUDY)`
- `src/features/review/SpeedReview.tsx`：`window.location.href = '/review'` → `navigate(ROUTES.REVIEW)`
- `src/pages/MaterialPage.tsx`：`navigate('/')` → `navigate(ROUTES.HOME)`

**注**：`SettingsPage.tsx` 中的 `window.location.reload()` 保留不变，该操作用于清除 localStorage 后重置应用状态，属于合法用例。
