- **8-10**：超出预期，会用且强烈推荐

## 六、基于使命/价值观的排期判断

### 产品使命
> "帮助死磕型学习者，把一个个概念真学懂、串起来，构建自己的心智格栅——像查理·芒格一样，在自己独有的知识体系中驰骋。"

### 判断原则
1. **P0（一期）**：必须修，否则目标用户的核心体验断裂
2. **P1（二期）**：极大提升体验，但缺少时用户仍可完成核心闭环
3. **P2（三期）**：锦上添花，有更好但没有也不影响核心价值
4. **Never**：与目标用户画像不符

### 最终排期

---

## 一期（P0，现在就要做的）

| 优先级 | 问题 | 依据 |
|--------|------|------|
| P0-1 | **开箱即用（内置默认API/演示模式自动降级）** | 18人提及，直接拦住所有非技术用户。核心用户虽然会配，但他们的朋友/学生不会 |
| P0-2 | **复习中心重做** | 17人提及，评分4.5是所有模块最低。复习是"夯砖石"的必要环节，不能不复习 |
| P0-3 | **PDF/网页素材导入** | 21人提及，核心用户的内容源复杂，不能纯靠粘贴。而且这不是后端能力，前端mammoth/PDF.js能解决 |
| P0-4 | **归档搜索/筛选** | 13人提及。概念100+后，没有搜索等于知识死档 |

### 一期具体规划

**P0-1 开箱即用**
- 实现方案：首次访问时自动检查 `zhika-settings` 是否存在 API Key
- 如果无 Key，提供入口：
  - 选项A：使用 Demo 模式（已有，流程走通即可）
  - 选项B：一键申请免费 API Token（需对接某个免费服务商，如 DeepSeek）
- 目标：用户从点击链接到第一次使用 AI 生成卡片，≤ 10 秒，零配置

**P0-2 复习中心重做**
- 当前问题：翻卡片看定义 → 低效
- 改造方案：
  - 新增"主动回忆"模式：只显示术语，让用户自己说出定义，然后对比
  - 新增"选择题"模式：AI 根据融合卡片生成 4 选 1
  - 新增"场景判断"模式：给出一个场景，问这个场景是否适用该概念
  - 评分系统：每次复习答对/答错记录，直接反馈"这个概念的薄弱点"
- 设计理念：死磕的人需要"验证自己是否真懂了"，不是"再看一遍"

**P0-3 PDF/网页导入**
- 使用 PDF.js 解析 PDF（纯前端，无后端）
- 使用 Readability 提取网页正文
- 微信公众号文章：用户复制链接，后端 fetch 内容（需 Serverless API）
- 目标：覆盖 80% 核心用户的素材来源

**P0-4 归档搜索**
- 按术语搜索（模糊匹配）
- 按领域/等级筛选
- 按材料来源筛选
- 按学习时间排序

## 七、决策备忘

### 这次不做的（与画像冲突）

| 功能 | 不做的原因 |
|------|-----------|
| 天梯榜/排行榜 | 死磕型学习者不在乎排名，他们在乎自己懂了多少 |
| 打卡签到/徽章系统 | 同上，非目标用户的动机 |
| 游戏化学习 | 与"下笨功夫"的理念冲突 |
| 内容创作社区 | 种子用户不需要，且运营成本高 |
| 视频课程/直播 | 太重了，偏离"概念学习"核心 |
| iOS/Android 原生APP | 先用PWA解决，原生APP后续评估ROI |

### 关键风险

1. **AI质量风险**：如果AI生成质量低（特别是法律/医学），核心用户会流失。解决方案：在专业领域增加"AI声明"和"人工精炼建议"
2. **数据安全风险**：核心用户投入大量精力构建知识体系，一旦数据丢失信任崩塌。云同步是二期必须
3. **定价模型风险**：如果按API消耗收费，重度用户成本会很高。建议：内容生产免费用（卡片生成），存储+同步付费
